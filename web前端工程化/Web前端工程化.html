  • 2.前端模块化的指引与实践
    • 什么是模块
      • 我们通常将具有特定功能的代码封装为模块
    • 为什么要使用模块
      • 作用域封装
      • 重用性
      • 解除耦合行
      • 按需加载
    • js模块发展史
      • script标签
      • 立即执行函数表达式
      • AMD define(模块名,模块依赖,函数/对象)
        • 第三个参数如果是函数,返回值(return)将定义的模块接口导出
        • 如果是对象,代表当前模块的导出值
        • 好处在于显示的表达每个模块所依赖的其他模块,并且模块定义也不会绑定到全局对象上,不必担心在别的地方被篡改
      • CommonJS与Node.js模块系统
        • 通过exports. 前缀导出,require函数引入
      • ES6 Module
        • export导出,import..from..导入
    • 动态加载模块
      • CommonJS的模块依赖是动态的
      • ES6 Module模块的依赖是静态的,在编译时确定的不是运行时确定的,在编译阶段就获得整个依赖树,从而进行代码静态分析层面的优化
      • CommonJs中require可以在if判断语句中使用,ES6 Module不可以
      • ES6 Module使用import()函数,返回一个Promise对象,实现动态加载,import(模块路径),then(module => {})
    • 模块打包原理简述
      • 模块打包:安装我们定义好的依赖树将模块合并成单一的文件,让浏览器按照预想好的依赖顺序去执行,这个过程通常叫做模块打包
      • 1.从入口文件开始分析整个应用的依赖树
      • 2.将每个依赖模块包装起来,并放到一个数组中等待调用
      • 3.实现模块加载的方法,并提供到模块执行的环境中,使得模块间可以互相调用
      • 4.将执行入口文件的逻辑放到一个立即执行函数表达式中
    • ES6 Module与浏览器
      • 1.对于不支持ES6 M的浏览器如何向下兼容,对于同一个页面来说要兼容两种浏览器就需要插入两种脚本,打包和未打包过的,目前来看这个问题不可避免
      • 仍然需要由工具来完成一些其它的构建任务,比如代码预编译,tree-shaking,代码压缩等,这些还没有办法直接在浏览器层面解决.
  • 3.关于包管理器你应该知道的
    • 什么是包管理器
      • 包管理器是一个可以让开发者便捷地获取代码和分发代码的工具.
    • npm特性一览
      • npm是js语言的包管理器
      • 初始化npm工程
        • mkdir npm-demo && cd npm-demo #新建并进入目录
        • npm init #初始化npm项目
        • package.json
          • name:当我们将包上传到npm平台上时,它类似于该报的id,不能和平台上已经有的包冲突.
          • version:这里的版本遵循npm特有的semver语法
          • main:这是包执行的入口,当这个包被别的地方引入的时候,引入的其实就是这个入口文件
          • script:npm提供的脚本命令功能,利用它可以实现很多开发和构建中的自动化命令
    • 仓库与依赖
      • 仓库是一个遵循npm特定包规范的站点,它提供API来让用户上传和下载包,获取包信息,已经管理用户账号
      • 设置淘宝镜像:npm config set registry https://registry.npm.taobao.org
      • npm install 从远程或者目标路径获取npm包
        • -S或者--save: 让刚刚安装的包记录在package.json的dependencies中,否则只是安装在本地的node_modules重,没有记录会导致不同环境下npm install的结果不一样
        • -D或者--save-dev: 指定当前环境是开发环境,将包的信息记录在devDependencies中
        • --only=prod：只会安装dependencies中的包
        • --only=dev：只会安装devDependencies中的包
        • 强制安装,不管是否安装过 npm install <packageName> --force
    • semver版本机制
      • npm5提供新的约束版本号的文件--package-lock.json,只能用在顶层包不能存在与依赖包中
      • npm install 执行时，node会先从package.json文件读取模块名称，node_modules目录中是否已经存在指定模块,存在就不重复安装,然后从package-lock.json文件中获取版本号，然后进行下载或者更新。所以node不会自动更新package.json文件中的模块。
      • npm install packagename(模块名称) #自动更新小版本号
      • npm install packagename@x.x.x #指定更新版本号，package-lock.json自动更新
      • 当package.json与package-lock.json都不存在是，执行npm install,node会重新生成package-lock.json文件，然后把node_modules中的模块信息全部计入lock文件中，但是不会生成package.json文件。可以通过npm init --yes来初始化该文件
    • 固定版本号,保障你的应用
    • Yarn带来的变革
      • yarn config set registry https://registry.npm.taobao.org
      • 替代npm客户端来管理原有的node_modules模块
      • 自动生成和维护版本号描述文件，初次执行生成yarn.lock文件，减少了解析semver与获取模块最新版本的时间
      • 在拉去包时采用并行操作，优化了请求队列，更高效的利用当前的网络资源
    • 使用包管理器的正确姿势
      • 使用npm5或Yarn来管理模块
      • 万能修复大法？rm -rf node_modules&&npm i
      • 不知道包怎么来的？试试yarn why
  • 4.使用Webpack构建应用及最新特性解析
    • 为什么选择Webpack
      • Gulp,Grunt属于构建流程管理工具,通过定义和执行任务来完成构建工作
      • Browserify,Webpack,Rollup属于打包工具,核心功能是把模块按照特定规则合并到一起,使浏览器可以执行.
      • Webpack支持AMD,CommonJS,ES6 Module多种模块,可以通过loader和plugin来进行预编译语言,代码压缩.优势在于代码按需加载.
    • 从零开始Webpack
      • 定义配置文件
      • Dev Server
      • 一切皆模块与loaders
      • 资源与压缩
      • 按需加载
    • 使用Webpack的构建特性
      • tree-shaking
      • scope-hoisting
